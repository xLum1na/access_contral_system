## 1. 引言

### 1.1 编写目的

本说明书旨在明确[项目名称]智能门禁系统的功能需求、性能指标、接口规范及安全要求，作为系统设计、开发、测试及验收的依据。本文档面向项目经理、系统架构师、软件开发人员、硬件实施方及最终用户代表。

### 1.2 项目背景

随着数字化转型的深入，传统刷卡/密码门禁已无法满足当前对**高安全性、无感通行、数据联动**的需求。本项目拟建设一套融合人脸识别、二维码等多种认证方式的AIoT智能门禁系统，实现人员权限的精细化管控、出入记录的实时追溯以及与安防/消防/考勤系统的深度联动。

### 1.3 术语定义

- **AIoT：** 人工智能物联网，指门禁终端具备边缘计算能力，支持本地人脸比对与活体检测。
- **多模态认证：** 支持人脸、扫码等两种及以上组合验证方式。
- **云边协同：** 核心数据与策略在云端管理，实时识别与开门逻辑在边缘设备（控制器/终端）执行，断网状态下仍可正常运行。

本系统是[智慧园区/智慧社区/办公大楼]综合安防管理平台的核心子系统，向上对接统一身份认证平台，向下管控各类电控锁具与通道闸机，横向联动视频监控、消防报警及访客系统。

- **客户端：** Web管理后台、iOS/Android移动APP、微信小程序。

## 3. 功能性需求

### 3.1 身份认证与通行管理

1. **多模态识别：** 支持人脸（1:N/N:1）、动态二维码认证。
2. **活体检测：** 人脸识别终端必须支持双目/3D结构光活体检测，有效防范照片、视频、面具攻击，误识率≤0.0001%。
3. **无感通行：** 识别速度≤0.3秒/人，支持多人同时抓拍与通行记录，高峰期通行效率≥30人/分钟。
4. **离线运行：** 网络中断时，前端设备应能存储不少于5万条人脸库和10万条通行记录，联网后自动同步。
5. **特殊通行：** 支持VIP迎宾模式、残疾人/老人关怀模式（延长开门时间）、胁迫密码报警。

### 3.2 权限与组织管理

1. **多级架构：** 支持集团-子公司-部门-班组无限级组织架构，支持按角色/标签批量授权。
2. **时段策略：** 支持自定义通行时段（如工作日9:00-18:00）、节假日策略、临时授权（精确到分钟）。
3. **区域管控：** 支持AB门互锁、防尾随、反潜回、多门联动等高级逻辑。
4. **黑白名单：** 支持实时下发黑名单，命中后禁止通行并触发声光告警及弹窗提示。

### 3.3 访客管理

1. **线上预约：** 访客通过小程序/H5提前预约，被访人或管理员审批后自动生成限时通行凭证（二维码/人脸）。
2. **现场登记：** 支持身份证OCR读取、人证比对核验、小票打印。
3. **轨迹追踪：** 访客仅可进入授权区域，超时或越界自动告警。

### 3.4 监控与告警

1. **实时状态：** 地图化展示所有门禁点开关状态、在线状态、最后通行信息。
2. **异常告警：** 支持门长时间未关、非法闯入、暴力破坏、设备掉线、多次认证失败等告警，支持短信/APP推送/声光联动。
3. **视频联动：** 告警触发时自动关联附近摄像头抓拍录像，并在管理平台弹窗显示。

### 3.5 数据与报表

1. **通行记录：** 完整记录人员、时间、地点、认证方式、抓拍照片、体温（如有）等信息，保存期限≥180天。
2. **统计分析：** 提供出勤统计、人流热力图、高峰时段分析、异常事件汇总等可视化报表，支持Excel/PDF导出。
3. **审计日志：** 所有管理员操作（增删改查、授权、开门）均需留痕，不可篡改。

### 3.6 系统集成与扩展

1. **消防联动：** 接收消防信号后，指定区域门禁自动强制释放，确保逃生通道畅通。
2. **电梯联动：** 刷脸/刷卡后自动呼叫电梯并点亮对应楼层权限。
3. **第三方对接：** 提供标准化RESTful API / MQTT接口，支持与HR系统（人员同步）、OA系统（审批流）、视频监控平台集成。

## 4. 非功能性需求

### 4.1 性能要求

- **并发能力：** 支持≥1000个终端同时在线，API响应时间≤500ms（P99）。
- **识别精度：** 人脸识别准确率≥99.9%，活体检测通过率≥99.5%。
- **可靠性：** 系统可用性≥99.95%，MTBF≥50,000小时。
- **容量：** 单设备支持人脸库≥50,000张，通行记录≥200,000条。

### 4.2 安全要求

- **数据安全：** 人脸特征值加密存储（AES-256），传输全程TLS加密；敏感个人信息脱敏展示；符合《个人信息保护法》及GB/T 35273要求。
- **访问控制：** 管理后台支持RBAC权限模型，强制双因素认证（2FA）。
- **隐私合规：** 采集人脸前需获得用户明示同意，提供便捷的删除/撤回同意渠道。
- **物理安全：** 室外设备IP65以上防护等级，防拆报警，宽温工作（-30℃~60℃）。

### 4.3 易用性与可维护性

- 管理界面简洁直观，关键操作3步内完成。
- 支持设备远程固件升级（OTA）、远程重启、远程配置下发。
- 提供完善的在线帮助文档与故障自诊断功能。

## 5. 约束与假设

### 5.1 约束条件

- 利旧要求：需兼容现有[品牌型号]门锁/闸机/控制器（如适用）。
- 网络限制：部分区域仅有RS485总线，需通过网关转换接入IP网络。
- 法规遵从：必须符合GB 50396-2007《出入口控制系统工程设计规范》及当地公安技防要求。

### 5.2 假设前提

- 甲方已提供准确的门点清单、平面图及强弱电点位。
- 现有网络基础设施满足门禁系统带宽与延迟要求。
- 相关人员信息（姓名、工号、照片）可由HR系统准确提供。

2. 连续7×24小时试运行无重大故障。
3. 第三方安全测评报告无高危/中危漏洞。
4. 提交完整交付物：竣工图纸、设备清单、操作手册、API文档、培训记录。
