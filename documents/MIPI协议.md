# MIPI协议

**MIPI协议是一套为移动设备内部组件互联而设计的、开放的接口标准集合。** 它的核心目标是通过标准化摄像头、显示屏等模块的接口，来降低设备设计的复杂度，并提升组件的兼容性和灵活性

## 1.MIPI 协议接口

MIPI有许多的协议接口，被使用最多的协议接口为为MIPI DSI、MIPI CSI等协议接口。

MIPI协议接口有以下优势：

* 高速数据传输：MIPi接口支持高速数据传输，最高可达数Gbps，满足移动设备对高分辨率、高刷新率显示的需求。
* 低功耗：MIPi接口采用低电压信号，降低功耗，延长移动设备的续航时间。
* 灵活性：MIPi接口支持多种数据通道和时钟频率，可以根据不同的应用场景进行配置。
* 兼容性：MIPi接口具有很好的兼容性，可以支持多种类型的显示设备和摄像头设备。



## 2.MIPI CSI

​	**MIPI CSI（Camera Serial Interface）是MIPI联盟为摄像头模块与处理器之间连接而制定的高速串行接口标准。** 作为当今移动设备、嵌入式视觉系统中应用最广泛的摄像头接口，它为图像传感器到应用处理器之间的数据传输提供了标准化、高带宽、低功耗的解决方案。

​	CSI-2采用通道可扩展（lane-scalable）设计，支持根据带宽需求灵活配置数据通道数量，典型配置为1至4个数据通道（Lane），部分实现可支持更多。单通道传输速率覆盖从80Mbps到9Gbps的广阔范围（具体取决于物理层版本、通道数量及是否启用均衡等高级功能）

### 2.1 分层架构

​	CSI-2采用清晰的三层架构设计，将物理层、协议层和应用层进行解耦，提升了系统的灵活性和兼容性。

| 层次       | 主要功能                                                     |
| ---------- | ------------------------------------------------------------ |
| **应用层** | 数据流的高级编码与解析，定义像素格式（RAW/YUV/RGB）及图像数据的组织方式 |
| **协议层** | 定义数据包格式、多数据流标识与合并机制、错误检测与纠正等     |
| **物理层** | 定义传输媒介、电气特性、时钟机制，具体由D-PHY或C-PHY实现     |

协议层内部又可细分为三个子层级：

- **像素/字节打包与解包**：将应用层的像素数据打包成字节流进行传输，接收端再解包还原。
- **底层协议（LLP）**：定义了长包（用于图像数据）和短包（用于控制信息）两种数据包结构，并规定了从低功耗状态退出（SoT）到传输结束返回低功耗状态（EoT）的完整流程。
- **通道分配**：支持根据带宽需求将数据分配到多个物理通道上传输，接收端负责合并并恢复原始序列。

### 2.2 工作模式

CSI-2物理层支持两种核心工作模式，通过动态切换实现性能与功耗的最优平衡。

- **高速模式（HS Mode）**：用于传输图像和视频等大数据量。采用低压差分信号（典型摆幅200mV），数据传输速率可达80Mbps至1Gbps（D-PHY）乃至更高（C-PHY可达2.5Gbps及以上），功耗相对较高但数据传输效率极高。
- **低功耗模式（LP Mode）**：主要用于传输控制指令、配置参数等小数据量场景。采用单端信号（电压摆幅约1.2V），数据速率较低（<10Mbps），但相应功耗非常低，有助于延长移动设备续航时间。

两种模式的有机配合，使得CSI-2总线在需要传输大量图像数据时可以高速运行，而在不需要大数据量传输时则切换到低功耗模式以节省能源。

### 2.3 数据包机制

CSI-2协议层基于数据包机制组织所有传输内容，所有数据均封装为特定格式的数据包。

**长数据包（Long Packet）**：用于传输实际的图像像素数据，长度可变。结构如下：

- 包头：包含虚拟通道标识（VC）、数据类型（DT）、数据长度计数、纠错码（ECC）等字段
- 有效数据：像素数据，格式由传感器配置决定（RAW8/RAW10/YUV422等）
- 包尾：包含校验码（CRC），用于数据完整性校验

**短数据包（Short Packet）**：用于传输控制信息或标记帧同步事件（如帧开始SOF、帧结束EOF），长度固定。它不携带有效载荷数据，仅通过包头信息即可完成控制功能。

### 2.4 数据结构与虚拟通道

CSI-2支持丰富的数据格式，以满足不同应用场景的需求：

- **RAW格式**：传感器原生输出格式（RAW8/RAW10/RAW12/RAW14/RAW24/RAW28），适合后续ISP进行图像处理
- **YUV格式**：经过压缩的亮色分离格式（YUV420/YUV422），带宽占用相对较低
- **RGB格式**：RGB565/RGB888等，直接用于显示，较少在CSI链路中传输

**虚拟通道（Virtual Channel）** 是CSI-2的另一重要特性，它允许一条物理链路同时承载多达32个独立的数据流。通过数据包头中的VC标识字段，接收端能够区分来自不同传感器的数据，从而实现多摄像头协同工作（如智能手机的多摄切换、深度感知等应用），提高了系统集成度。

### 2.5 物理层

CSI-2可运行在多种物理层之上，包括D-PHY、C-PHY、A-PHY等，设计者可根据应用场景灵活选择最适合的物理层方案。

- **D-PHY**：采用一对差分时钟线和1至多条差分数据线进行串行传输。成熟稳定，应用最广，单通道速率最高约4.5Gbps（D-PHY v2.0/v3.0可达更高）。
- **C-PHY**：采用三相三线嵌入式时钟编码技术，每个符号可携带约2.28比特信息，在同等引脚数下可获得更高的带宽效率。v1.0版本单通道可达5.7Gbps。
- **A-PHY**：专为汽车电子设计的长距离SerDes接口，传输距离可达15米，满足车载ADAS等应用对长距离、高抗干扰能力的需求。

CSI-2协议原生支持C-PHY、D-PHY或组合C/D-PHY配置，允许厂商根据设计需求灵活选择。物理层之上还定义了摄像机控制接口（CCI），双向兼容I2C标准，用于传感器的参数配置和控制。

### 2.6 可靠性机制

SI-2规范内置了完善的错误检测与纠正机制，确保图像数据传输的可靠性：

- **包头纠错码（ECC）**：可检测并纠正包头中的单比特错误，保护关键控制信息
- **包尾循环冗余校验（CRC）**：对有效数据段进行完整性校验，检测传输过程中的数据错误
- **重传协议**：在检测到错误时触发重传机制，确保数据准确到达接收端
- **精确时序校准**：采用非对称传输模式和精确的时序校准机制，确保数据在高速传输过程中的完整性



## 3.MIPI DSI

​	**MIPI DSI（Display Serial Interface）是MIPI联盟为显示模块与应用处理器之间连接而制定的高速串行接口标准。** 它旨在以串行方式高效传输像素数据和控制指令，成为智能手机、平板电脑、可穿戴设备、车载显示屏及AR/VR设备等领域的主流显示接口。

​	DSI标准由MIPI联盟的Display工作组制定，其核心目标是通过串行化设计，降低移动设备中显示控制器与外设之间的连接复杂度，同时实现高带宽与低功耗的统一。它将传统并行显示接口中所需的多种控制信号（如VSYNC、HSYNC、DE等）进行串行化处理，并通过统一的差分信号线进行传输，大大减少了引脚数量，降低了PCB设计难度。

###  3.1 分层架构

​	与同属于MIPI家族的CSI类似，DSI也采用分层架构设计，包括物理层（PHY Layer）、通道管理层（Lane Management Layer）、底层协议层（Low Level Protocol Layer）和应用层（Application  Layer），各层职责清晰，具有良好的扩展性和兼容性。DSI规范兼容DPI、DBI和DCS等传统显示接口概念，使得从传统并行接口过渡到串行DSI接口的迁移路径更加平滑。

### 3.2 工作模式

DSI支持两种核心工作模式，以适应不同的显示传输场景和功耗需求。

**高速模式**：用于传输实际的像素数据和视频流，所有数据通道均可用于单向高速传输。该模式下数据传输速率可达数Gbps（取决于物理层版本），支持高分辨率、高刷新率的显示输出。在高速传输过程中，专用的时钟通道负责传输同步时钟信号，确保数据接收的时序准确性。

**低功耗模式**：主要用于传输控制命令（如DCS指令集）、状态查询或进行低速双向通信。值得注意的是，**只有第一个数据通道（Lane 0）支持低速双向传输**，从属端（如显示屏模块）的状态信息、像素数据等可以通过该通道返回到主机端。其他数据通道在低功耗模式下仅支持单向传输（主机到从属端）或处于空闲状态。

### 3.3 通道结构与带宽

DSI接口通常采用1条差分时钟通道（Clock Lane）和1至4条差分数据通道（Data Lane）的基本结构。通道数量的选择直接决定了显示接口的可用带宽。

- 单条数据通道速率可达数Gbps（D-PHY通常支持1.5Gbps至4.5Gbps，C-PHY可达更高）
- 多通道并行可成倍提升总带宽：例如4条数据通道配置下，总输入带宽可达12Gbps以上
- DSI-2进一步引入C-PHY物理层支持，在同等引脚数下可获得更高的带宽效率

通用硬件接口引脚数通常为19（D-PHY）或22（C-PHY），支持RGB/YUV/RAW等多种显示数据格式，最高可支持3840×2160@60Hz甚至更高的显示分辨率。

### 3.4 视频模式（Video Mode）

DSI在高速模式下定义了多种视频传输模式，以适应不同显示刷新策略的需求。主要包括以下三种视频模式。

- **Non-Burst Mode with Sync Pulses**：最传统的同步模式，在每个视频帧/行的开始位置发送独立的同步脉冲信号，时序要求精确但带宽利用率相对较低。
- **Non-Burst Mode with Sync Events**：采用更紧凑的事件信号替代独立的同步脉冲，减少冗余信号的传输，带宽利用率有所提升。
- **Burst Mode**：将像素数据以"爆发"方式集中传输，数据填充后进入空闲状态，带宽利用率最高，适用于需要将节省的带宽分配给其他子系统（如CPU或GPU）的场景。

### 3.5 命令模式与DCS指令集

除了视频模式外，DSI还定义了**命令模式（Command Mode）**，主要面向带有本地帧缓冲（GRAM）的显示模块。在命令模式下，主机无需持续刷新显示屏，仅在需要更新显示内容时发送数据即可，这有助于降低系统功耗。

DSI规范集成了**DCS（Display Command Set）** 标准化指令集，用于对显示模块进行控制和配置。DCS指令涵盖了显示初始化、电源管理、亮度调节、睡眠模式进入/退出等标准化操作。通过统一的指令集，不同厂商的显示模块可以在软件层面得到一致的控制方式，提高了系统集成效率。

### 3.6 物理层

DSI在物理层方面经历了从仅支持D-PHY到支持D-PHY和C-PHY双物理层的演进过程。

- **DSI（早期版本）**：仅支持D-PHY物理层，采用差分时钟和数据线结构，成熟稳定，应用广泛。
- **DSI-2（v1.0及后续版本）**：引入C-PHY物理层支持，采用三相三线嵌入式时钟编码技术，可在同等引脚数下获得更高的带宽效率，原生支持4K/8K超高清显示。DSI-2 v2.0进一步添加了从高分辨率视频模式到低延迟图形界面的无缝切换能力，并兼容所有以前版本的规范。v2.1版本则引入了增强的PPI训练功能，进一步优化了物理层接口时序。

### 3.7 双向通信与多从属端管理

DSI的另一重要特性是对双向通信的支持。虽然大部分显示链路的数据流向是主机到显示屏的单向传输，但DSI规范允许主机通过Lane 0从显示屏模块读取状态信息、像素数据、触控信息等。这种双向能力使得显示屏模块可以向主机反馈自身状态（如温度、功耗等），也为主机端实现更智能的显示管理提供了可能性。

此外，一个DSI主机端可允许同时与多个从属端（如多个显示屏模块）进行通信，通过虚拟通道管理和地址识别机制实现多屏协同显示或分屏显示等高级功能。
